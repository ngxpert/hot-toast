🔗 Router integration

Call toast.onRouteChange(messages) once (typically in a component constructor or a root service) to automatically show a toast on every Angular Router event. Keys in messages are Router event class names — use NavigationStart.name etc. for type safety. Values are either a plain string / template / component (displayed as a blank toast) or a full options object with message and any ToastOptions fields including an optional type.

While the previous toast is still visible, subsequent router events update it in place rather than stacking new ones. The subscription is automatically torn down when the calling component is destroyed if onRouteChange is called from an injection context (e.g. a constructor). Otherwise the returned Subscription can be managed manually.

Basic usage

Pass plain strings for the events you care about. The subscription is tied to the component's lifetime automatically when called from the constructor.

Full options

Replace any plain string with a RouteChangeToastOptions object to control the toast type, duration, style and every other ToastOptions field per-event. All Angular Router events are supported as keys.

Manual cleanup

When calling outside a constructor (e.g. ngOnInit), pass a DestroyRef as the second argument or unsubscribe from the returned Subscription yourself.
